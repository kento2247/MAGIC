## Proposed Method

We propose Affordance RAG, a zero-shot hierarchical multimodal retrieval framework for OVMM.  
Fig.~\ref{fig:framework} shows the overview of the proposed method.

To understand what kind of robot affordances are executable in an environment, we introduce **Affordance-Aware Embodied Memory (Affordance Mem)**, constructed from observed images through pre-exploration, as shown in Fig.~\ref{fig:framework} (a).

Our method utilizes Affordance Mem as a real-world database for RAG to identify both the target object and the receptacle, as shown in Fig.~\ref{fig:framework} (b).

While our primary focus is on OVMM, the proposed Affordance Mem is broadly applicable to other embodied reasoning tasks such as EQA~\cite{Majumdar_2024_CVPR,yang20243dmem3dscenememory}.

---

### Affordance-Aware Embodied Memory

For robust OVMM, it is important to identify the most suitable option in terms of robot affordance when multiple valid candidates exist.

To address this limitation, we propose a hierarchical embodied memory spanning multiple levels of nodes—from robot affordance, instance, view, zone, and area to building—across levels $1$ through $N$.

As illustrated in Fig.~\ref{fig:framework} (a), Affordance Mem is constructed sequentially in a bottom-up manner using three components:

1. **Affordance Proposal**: predicts instance-level affordances  
2. **Multi-Level Representation**: extracts view-level semantics  
3. **Area Summarization**: aggregates multi-view features  

---

#### **Affordance Proposer (Level 1–2)**

In this module, we apply object-centric visual prompting to $\bm{x}_\mathrm{img}^{(j)}$ and feed the resulting image into VLMs (e.g., GPT-4o~\cite{gpt_4o}) to extract instance-level representations and robot affordances, which are stored as level 2 and level 1 nodes, respectively.

This enables the extraction of descriptive expressions (e.g., *"red metal mug," "antique wooden side table with vertical slats"*) beyond category-level nodes (e.g., *"cup," "table"*) typically obtained via object detection or semantic segmentation~\cite{momallm24,Werby-RSS-24}.

Specifically:
- We use SEEM~\cite{zou24neurips} to generate segmentation masks  
- We create a visual prompt with indexed regions  
- Both original and prompted images are fed into the VLM  

Although prior works~\cite{yang2023arxiv,relaxformer} explored similar captioning strategies, our method extends this to **affordance-aware memory construction**.

The output is a set of instance-affordance triplets.

Since this module is prompt-based, it can be easily extended to other actions (e.g., *open*, *close*) without architectural changes.

---

#### **Multi-Level Representation (Level 3)**

This module constructs a multi-level representation for $\bm{x}_\mathrm{img}^{(j)}$ by combining:

- **High-level regional semantics**
- **Low-level visual semantics**

These are stored as level 3 nodes.

Specifically:

- Regional semantics:
  
  $$
  \bm{l}^{(3,j)}_\mathrm{s} \in \mathbb{R}^{d_\mathrm{t}}
  $$

  obtained from VLM-generated descriptions encoded via a text encoder (e.g., text-embedding-3-large~\cite{txtembedding3large})

- Visual semantics:

  $$
  \bm{v}^{(3,j)} \in \mathbb{R}^{d_\mathrm{m}}
  $$

  extracted using a multimodal encoder (e.g., BEiT-3~\cite{beit3})

Here, $d_\mathrm{t}$ and $d_\mathrm{m}$ denote embedding dimensions.

The fusion strategy is described in Sec.~\ref{sec:mlf}.  
These descriptions are also used for hierarchical aggregation in the Area Summarizer.

---

#### **Area Summarizer (Level $N$)**

This module performs agglomerative clustering to construct higher-level nodes.

For each level $i \ge 3$, level $(i+1)$ nodes are generated by clustering nodes at level $i$.

Unlike conventional models that process images independently, this module captures **out-of-view context**, which is crucial for instructions such as:

- *"towel near the sink"*
- *"in the bedroom"*

Clustering is based on:

- Euclidean distance (spatial)
- Cosine similarity (semantic)

The procedure is repeated recursively:

1. Cluster nodes  
2. Generate cluster summaries using an LLM  
3. Encode summaries:

   $$
   \bm{l}^{(i+1,j)}_\mathrm{s} \in \mathbb{R}^{d_\mathrm{t}}
   $$

This enables hierarchical and semantically grounded environment understanding.

---

### Hierarchical Multimodal Retrieval

Fig.~\ref{fig:framework} (b) illustrates retrieval over Affordance Mem given a natural language instruction.

The process proceeds in a **top-down manner** with three stages:

1. Recursive Top-Down Traversal  
2. Multi-Level Fusion  
3. Affordance-Aware Reranking  

---

#### **Recursive Top-Down Traversal**

We traverse from level $N$ to level 3 by selecting top-$K_\mathrm{b}$ nodes based on:

$$
s^{(i,j)} = \mathrm{sim}(\bm{l}_\mathrm{t}, \bm{l}^{(i,j)}_\mathrm{s})
$$

where:

- $\mathrm{sim}(\cdot)$: cosine similarity  
- $\bm{l}_\mathrm{t} \in \mathbb{R}^{d_\mathrm{t}}$: instruction embedding  
- $\bm{l}^{(i,j)}_\mathrm{s} \in \mathbb{R}^{d_\mathrm{t}}$: node embedding  

This enables coarse-to-fine filtering using regional semantics.

Compared to LLM-based selection~\cite{Xie2024EmbodiedRAGGN,wang2025navraggeneratinguserdemand}, this approach:

- scales better to large candidate sets  
- enables faster inference via precomputed embeddings  

Output:

$$
\mathcal{S} \subseteq \{ (\bm{v}^{(3,j)}, \bm{l}_\mathrm{s}^{(3,j)}) \}
$$

---

#### **Multi-Level Fusion**
<label id="sec:mlf"></label>

We compute a multimodal similarity score:

$$
s_\text{mlf}^{(3,j)} =
\alpha \cdot \mathrm{sim}(\bm{l}_\mathrm{t}, \bm{l}^{(3,j)}_\mathrm{s})
+
(1 - \alpha) \cdot \mathrm{sim}(\bm{l}_\mathrm{m}, \bm{v}^{(3,j)})
$$

where:

- $\alpha \in [0,1]$: weighting parameter  
- $\bm{l}_\mathrm{m} \in \mathbb{R}^{d_\mathrm{m}}$: multimodal text embedding  

This fuses:

- regional semantics (hierarchical memory)
- visual semantics (multimodal encoder)

Unlike prior work that uses only one modality, this enables **complementary retrieval**.

---

#### **Affordance-Aware Reranking**

We rerank top-$K_\mathrm{r}$ nodes using affordance information $\{\mathcal{A}^{(j)}\}$.

The process consists of:

1. **Affordance Prefiltering**  
   - filter by actions (*pick*, *place*)

2. **Descriptive Instance Retrieval**  
   - LLM scores instance descriptions  

3. **Affordance Score Reranking (ASR)**  
   - refine top-$K_\mathrm{f}$ nodes  

Finally, view nodes are reranked by prioritizing those containing selected instances.

The process is performed separately for:

- target object → $\hat{Y}_\mathrm{targ}$  
- receptacle → $\hat{Y}_\mathrm{rec}$
